# Contributing to arpass-spec

[日本語版は下にあります](#日本語版)

Thank you for your interest in contributing to arpass-spec.

This repository is the **public spec mirror** of [Arpass](https://arpass.io), a blockchain-backed password manager by [Technoblest](https://technoblest.com). It serves three audiences:

- **Security researchers** who want to audit the cryptographic design
- **Auditors** who need to verify the deployed bundle hashes
- **Developers** who want to study or self-host an alternate Arpass client

## Two kinds of files in this repo

This repo contains two categories of files. **Where you can submit a PR depends on which category the file falls into.**

### 🟢 Manually maintained — PRs welcome

PRs to these files are reviewed and merged by maintainers:

| File / directory | What it is |
|---|---|
| `README.md`, `README.en.md` | Public-facing intro to arpass-spec |
| `docs/envelope-v5.md`, `docs/envelope-v4.md` | Cryptographic envelope specification |
| `docs/crypto-rationale.md` | Algorithm choice rationale |
| `docs/arweave-tags.md` | Arweave tag schema |
| `docs/en/*` | English versions of the above |
| `lib/*.js` | Snapshot of cryptographic client code (see note below) |
| `LICENSE` | AGPL-3.0 license text |
| `CONTRIBUTING.md` (this file) | Contribution guide |

**Good PRs to send:**
- Typo fixes and translation improvements
- Clarifications to ambiguous spec language
- Additional examples or counter-examples in `docs/`
- Errata in cryptographic descriptions
- New language translations (e.g., `docs/zh/`, `docs/ko/`)

**Note on `lib/*.js`**: these are *snapshots* of the active client code shipped from the private repo. Substantive code changes go upstream; this repo accepts only documentation-style PRs (comment improvements, additional inline examples). Behavioral changes will be marked "needs upstream review" and the maintainer will port them.

### 🔴 Auto-generated mirror — PRs will be closed

Do **not** submit PRs to these files. They are automatically synced from the private upstream repository on every release; any direct edit will be overwritten on the next sync.

| File | Source of truth |
|---|---|
| `SECURITY.md` | Private upstream — security policy authored there |
| `BUILD-HASHES.md` | Generated by the mirror workflow |

**If you found an issue in `SECURITY.md`** (typo, broken link, outdated info), please open a GitHub Issue here describing it. We will fix it upstream and the next sync will propagate the fix.

**If you found a hash mismatch in `BUILD-HASHES.md`** vs. the live `arpass.io` deployment, that is a **potential security incident**. Report it via the Security channel:

- Open a [Private Security Advisory](https://github.com/technoblest/arpass-spec/security/advisories/new)
- Or email `security@arpass.io` (PGP key in `SECURITY.md`)

## How to send a PR

1. Fork this repo
2. Create a branch from `main`
3. Make your changes
4. Open a PR to `technoblest/arpass-spec:main`
5. CI will run lint checks if applicable
6. A maintainer will review within 1 week (best effort)

## Reporting security issues

Please do **not** open public issues or PRs for security-sensitive matters. See [SECURITY.md](./SECURITY.md) for the proper channels.

## このリポジトリには 2 種類のファイルがあります

提出可能な PR の種類が、ファイルの分類によって異なります。

### 🟢 手動管理 — PR 歓迎

これらのファイルへの PR はメンテナがレビュー・merge します:

| ファイル / ディレクトリ | 内容 |
|---|---|
| `README.md`, `README.en.md` | arpass-spec の入口 |
| `docs/envelope-v5.md`, `docs/envelope-v4.md` | 暗号エンベロープ仕様 |
| `docs/crypto-rationale.md` | アルゴリズム選定の根拠 |
| `docs/arweave-tags.md` | Arweave タグスキーマ |
| `docs/en/*` | 上記の英語版 |
| `lib/*.js` | 暗号クライアントコードのスナップショット (下記注釈) |
| `LICENSE` | AGPL-3.0 ライセンス |
| `CONTRIBUTING.md` (本ファイル) | コントリビューションガイド |

**歓迎する PR の例:**
- 誤字脱字の修正、翻訳の改善
- 曖昧な仕様文言の明確化
- `docs/` への追加例・反例
- 暗号記述の正誤訂正
- 新言語への翻訳追加 (例: `docs/zh/`, `docs/ko/`)

**`lib/*.js` について**: private リポジトリから配信されている実コードの *スナップショット* です。動作変更は上流側で対応するため、このリポジトリではドキュメンテーション系 PR (コメント改善・補助コメント追加) のみ受け付けます。動作変更を含む PR は「要上流レビュー」とラベル付けし、メンテナが上流へ移植します。

### 🔴 自動ミラー — PR は close されます

以下のファイルへの PR は提出**しないでください**。リリースごとに private リポジトリから自動同期されており、直接の編集は次の同期で上書きされます。

| ファイル | 原本 |
|---|---|
| `SECURITY.md` | private 上流 — セキュリティ方針はそこで管理 |
| `BUILD-HASHES.md` | mirror ワークフローが自動生成 |

**`SECURITY.md` に問題を見つけた場合** (誤字、リンク切れ、情報の古さ等) は、このリポジトリで **GitHub Issue** を立てて報告してください。上流側で修正し、次回の同期で反映されます。

**`BUILD-HASHES.md` のハッシュと `arpass.io` の実配信ファイルが一致しない場合**、それは **セキュリティインシデントの可能性** があります。以下のセキュリティ窓口へご連絡ください:

- [Private Security Advisory](https://github.com/technoblest/arpass-spec/security/advisories/new) を開く
- または `security@arpass.io` へメール (PGP 公開鍵は `SECURITY.md` 参照)

## PR の送り方

1. このリポジトリを fork
2. `main` からブランチを作成
3. 変更を加える
4. `technoblest/arpass-spec:main` への PR を開く
5. CI が必要に応じて lint チェックを実行
6. メンテナが 1 週間以内にレビュー (best effort)

## セキュリティ問題の報告

セキュリティに関する問題は **public issue / PR で報告しないでください**。適切な経路は [SECURITY.md](./SECURITY.md) を参照してください。
